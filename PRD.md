## 1. 产品概述

### 1.1 产品定位

**self-leveling（我独自升级）** 是一款将个人目标达成转化为游戏化体验的Web应用。它通过AI驱动的任务规划、打怪升级的积分系统和游戏化激励机制，帮助用户以更有趣、更可持续的方式完成个人目标。

### 1.2 核心价值主张

> "让成长变成一场冒险，把目标变成可以打败的怪物"

### 1.3 产品Slogan

**"我独自升级"** —— 强调个人成长旅程的独特性和自主性

### 1.4 目标用户

| 用户群体 | 画像描述 |
|---------|---------|
| 学生党 | 需要管理学习计划、备考、课外技能学习 |
| 职场人士 | 自我提升、技能升级、职业发展规划 |
| 自由职业者 | 项目目标管理、个人品牌建设 |
| 习惯养成者 | 想要建立好习惯、戒除坏习惯 |

### 1.5 核心概念

```
目标 (Goal) → AI规划 → 里程碑 (Milestone) + 每日任务 (Task)
                ↓
           游戏化升级系统
                ↓
        积分奖励 ← 完成任务
                ↓
        跳过能力 = 消耗积分
```

## 2. 用户故事

### 2.1 核心用户故事

#### US-001: 创建目标
**作为** 一个想要提升自己的用户  
**我想要** 输入我想达成的目标  
**以便** 获得AI为我定制化的任务规划  

#### US-002: 接收每日任务
**作为** 一个有目标在身的用户  
**我想要** 每天收到专属的任务提醒  
**以便** 知道自己今天需要做什么  

#### US-003: 完成任务获得积分
**作为** 一个努力完成任务的用户  
**我想要** 每完成一个任务获得相应积分  
**以便** 看到自己的努力被量化并获得成就感  

#### US-004: 使用积分跳过任务
**作为** 一个积分充足的用户  
**我想要** 用积分跳过某些任务  
**以便** 在特殊情况下灵活调整计划  

#### US-005: 追踪里程碑进度
**作为** 一个长期参与者  
**我想要** 看到自己在各里程碑的进度  
**以便** 保持动力并了解整体进度  

### 2.2 扩展用户故事

#### US-006: 查看积分历史
**作为** 一个用户  
**我想要** 查看积分获取和消耗的详细历史  
**以便** 了解自己的表现趋势  

#### US-007: 任务提醒通知
**作为** 一个忙碌的用户  
**我想要** 收到任务开始的提醒  
**以便** 不会忘记今日任务  

#### US-008: 暂停/恢复目标
**作为** 一个可能会遇到意外情况的用户  
**我想要** 临时暂停目标计划  
**以便** 不会因为中断而放弃整个目标

## 3. 功能需求清单

### 3.1 MVP阶段 (Must Have)

#### F-001: 用户注册与登录
- 邮箱/密码注册和登录
- Session管理
- 密码重置功能

#### F-002: 目标管理
- 创建目标（名称、描述、期望完成时间）
- 查看目标列表
- 编辑目标
- 删除目标
- 目标状态（进行中/已完成/已暂停）

#### F-003: AI任务规划引擎
- 调用AI API分析目标
- 生成里程碑划分
- 生成每日任务列表
- 为每个任务分配积分值
- 任务描述生成

#### F-004: 每日任务系统
- 每日任务展示
- 任务完成标记
- 今日任务概览
- 任务详情查看

#### F-005: 积分系统
- 完成任务获得积分
- 查看当前积分余额
- 积分历史记录
- 使用积分跳过任务
- 跳过任务消耗积分计算

#### F-006: 进度追踪
- 目标完成百分比
- 里程碑进度展示
- 已完成任务统计

### 3.2 完整版功能 (Should Have)

#### F-007: 任务跳过功能
- 跳过确认弹窗
- 跳过原因记录（可选）
- 跳过对进度的影响提示

#### F-008: AI重新规划
- 手动触发AI重新生成任务
- 任务生成历史记录

#### F-009: 数据统计面板
- 积分趋势图
- 任务完成率
- 连续完成任务天数
- 月度/周度报告

#### F-010: 通知系统
- 每日任务提醒（邮件/Push）
- 里程碑达成通知
- 积分变动通知

### 3.3 优化版功能 (Nice to Have)

#### F-011: 游戏化元素
- 等级系统（根据累计积分）
- 成就徽章
- 排行榜
- 任务连击奖励

#### F-012: 社区功能
- 目标分享
- 好友系统
- 任务挑战

#### F-013: 多语言支持
- 中文/英文界面
- 任务内容多语言生成

## 4. 非功能需求

### 4.1 性能要求
| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2秒 |
| AI任务生成时间 | < 10秒 |
| API响应时间 | < 500ms |
| 并发用户支持 | 100+ 同时在线 |

### 4.2 安全要求
- 用户密码加密存储（bcrypt）
- Session token安全传输
- API请求频率限制
- 敏感数据脱敏
- HTTPS强制使用

### 4.3 可用性要求
- 响应式设计（桌面/平板/手机）
- 浏览器兼容：Chrome, Firefox, Safari, Edge最新两个版本
- 断网情况下的基本体验（缓存已加载数据）

### 4.4 可维护性
- 模块化代码架构
- API文档完整
- 日志系统完善
- 错误追踪机制

### 4.5 合规性
- GDPR数据保护（欧盟用户）
- 用户数据导出功能
- 账户删除功能

## 5. 技术建议

### 5.1 技术栈建议

#### 前端
| 技术 | 建议 | 理由 |
|------|------|------|
| 框架 | Next.js / React | 生态丰富，SSR支持 |
| UI库 | Tailwind CSS + shadcn/ui | 现代化、快速开发 |
| 状态管理 | Zustand / React Query | 轻量级、适合数据获取 |
| 动画 | Framer Motion | 流畅的游戏化动画 |

#### 后端
| 技术 | 建议 | 理由 |
|------|------|------|
| 框架 | Next.js API Routes / Express | 与前端统一 |
| 数据库 | PostgreSQL | 关系型数据，稳定性 |
| ORM | Prisma | 类型安全、开发效率 |
| AI API | OpenAI GPT-4 / Claude | 成熟的AI能力 |

#### 部署
| 组件 | 建议 | 理由 |
|------|------|------|
| 托管 | Vercel | Next.js官方支持，无缝集成 |
| 数据库 | Supabase / Neon | PostgreSQL即服务 |
| 缓存 | Redis | 会话和实时数据 |
| 文件存储 | S3 / R2 | 用户上传文件 |

### 5.2 系统架构

```
┌─────────────────┐     ┌─────────────────┐
│    前端 (Web)   │────▶│   Next.js API   │
│   Next.js App   │     │    Routes        │
└─────────────────┘     └────────┬────────┘
                                 │
        ┌────────────────────────┼────────────────────────┐
        │                        │                        │
        ▼                        ▼                        ▼
┌───────────────┐     ┌─────────────────┐     ┌───────────────┐
│  PostgreSQL   │     │     Redis       │     │   AI API      │
│   (Prisma)    │     │   (缓存/会话)    │     │  (OpenAI)     │
└───────────────┘     └─────────────────┘     └───────────────┘
```

### 5.3 数据模型

#### User
```
- id: UUID
- email: String (unique)
- passwordHash: String
- username: String
- totalPoints: Integer
- level: Integer
- createdAt: DateTime
- updatedAt: DateTime
```

#### Goal
```
- id: UUID
- userId: UUID (FK)
- title: String
- description: Text
- targetDate: DateTime
- status: Enum (active/paused/completed)
- totalPoints: Integer
- currentProgress: Integer
- createdAt: DateTime
- updatedAt: DateTime
```

#### Milestone
```
- id: UUID
- goalId: UUID (FK)
- title: String
- description: Text
- order: Integer
- targetDate: DateTime
- isCompleted: Boolean
- completedAt: DateTime?
```

#### Task
```
- id: UUID
- milestoneId: UUID (FK)
- title: String
- description: Text
- points: Integer
- plannedDate: Date
- status: Enum (pending/completed/skipped)
- completedAt: DateTime?
- skippedAt: DateTime?
```

#### PointTransaction
```
- id: UUID
- userId: UUID (FK)
- taskId: UUID? (FK)
- amount: Integer
- type: Enum (earned/spent)
- description: String
- createdAt: DateTime
```

## 7. 风险评估

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| AI任务质量不稳定 | 高 | 提供手动调整入口；收集用户反馈优化提示词 |
| 用户初期流失 | 高 | 简化 onboarding；快速展示价值 |
| API成本控制 | 中 | 设置积分上限；任务数量限制 |
| 数据安全 | 中 | 加密存储；定期安全审计 |
